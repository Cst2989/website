Clustere Kubernetes

Kubernetes coordoneaza un cluster cu disponibilitate ridicată de computere care sunt conectate să funcționeze ca o singură unitate. Abstractizările din Kubernetes îți permit să publici aplicații containerizate într-un cluster fără să fie legate în mod specific de o mașină anume. Pentru a face uz de acest model de publicare, aplicațiile au nevoie să fie împachetate într-un mod care le decuplează de gazde individuale: au nevoie să fie containerizate. Aplicațiile containerizate sunt mai flexibile și disponibile decât în modelele de publicare anterioare, în care aplicațiile earu instalate direct pe mașini specifice ca pachete profund integrate în gazdă. Kubernetes automatizează distribuția și programarea aplicațiilor containerizate într-un cluster într-un mod mai eficient. Kubernetes este o platformă open-source și este pregătită pentru producție.

Un cluster Kubernetes este format din două tipuri de resurse:

  • Masterul coordonează clusterul
  • Nodurile sunt muncitorii care rulează aplicațiile

Rezumat:

  • Cluster Kubernetes
  • Minikube

Kubernetes este o platformă open-source, pregatită de producție care orecheastrează plasarea (programarea) și execuția containerelor de aplicații în și pe mai multe computere dintr-un cluster.


Diagrama Clusterului


Masterul este responsabil de administrarea clusterului. Masterul coordoneaza toate activitățile din clusterul tau, precum programarea aplicațiilor, menținerea stării dorite a aplicațiilor, scalarea aplicațiilor, și publicarea de actualizări.

Un nod este o mașină virtuală sau un computer fizic care servește ca mașină de lucru într-un cluster Kubernetes.. Fiecare nod are un Kubelet, care este un agent de administrare a nodului și de comunicare cu masterul Kubernetes. Nodul ar trebuie dealtfel să aibă unelte pentru manipularea operațiunilor containerului, precum Docker sau rkt. Un cluster Kubernetes care gestionează traficul de producție ar trebui să aiba minim trei noduri.

Resursele de tip Master gestionează clusterul și nodurile care sunt folosite pentru găzduirea și rularea aplicațiilor.

Când publici o aplicație în Kubernetes, îi spui masterului să pornească containerele aplicațîei. Masterul programarează containerele să ruleze pe nodurile clusterului. Nodurile comunică cu masterul folosind API-ul Kubernetes, pe care masterul îl expune. Utilizatorii finali pot de asemenea să folosească API-ul Kubernetes direct pentru a interacționa cu clusterul

Un cluster Kubernetes poate fi implementat pe mașini fizice sau virtuale. Pentru a începe dezvoltarea pe Kubernetes, poți folosi Minikube. Minikube este o implementare simplă a Kubernetes, care creează o mașină virtuala pe mașina ta locală și publică un cluster simplu care conține un singur nod. Minikube este disponibil pentru Linux, macOS și sisteme Windows. Clientul de terminal Minikube oferă operații de bază de bootstrapping pentru lucrul cu clusterul tău, incluzând pornirea, oprirea, vizualizarea stării și ștergerea. Cu toate acesta, pentru acest tutorial, vei utiliza un terminal online furnizat cu Minikube preinstalat.
